# El SolCapital Trading Dashboard

This project is a real-time trading dashboard powered by a Solace event mesh and enhanced with AI-driven insights. It consists of two main components: a web-based dashboard for visualizing stock data and a backend connector that integrates with large language models (LLMs) to provide trading signals.

## Components

### 1. Dashboard

The `dashboard` is a modern, responsive web application built with React, TypeScript, and Vite. It provides the following features:

-   **Real-time Stock Ticker**: Displays live updates for selected stocks.
-   **Market Overview**: Shows a dynamic panel of major market symbols.
-   **Advanced Filtering**: Allows users to subscribe to real-time data by stock, exchange, or country using wildcard topics.
-   **AI-Generated Insights**: Shows trading signals and the latest relevant tweets for stocks, generated by the backend AI connector.
-   **Interactive Configuration**: A comprehensive panel to manage connections to the Solace event mesh and control data feeds.

### 2. Solace AI Connector

The `solace-ai-connector` is a Python-based backend service that connects to the Solace event mesh. Its primary responsibilities are:

-   **LLM Integration**: Listens for specific events (like stock ticks or news) and uses a configured Large Language Model (e.g., from OpenAI, Claude, etc.) to generate analysis, such as trading signals or sentiment analysis from tweets.
-   **Event Publishing**: Publishes the AI-generated insights back onto the Solace event mesh for the dashboard and other services to consume.
-   **Dynamic Configuration**: The connector is configured at runtime using environment variables, ensuring that no sensitive credentials are hardcoded.

## Running the Project with Docker

The entire application is containerized and can be easily run using Docker and Docker Compose.

### Prerequisites

-   Docker Desktop installed and running.
-   A `.env` file created in the root of the `trading_dashboard_demo` directory.

### Step 1: Configure Your Environment

Before launching the application, you must provide your service credentials in a file named `.env`.

1.  Create a file named `.env` in the same directory as the `docker-compose.yaml` file.
2.  Add the following variables to the file, replacing the placeholder values with your actual credentials:

    ```env
    # Solace Broker Connection Details
    SOLACE_BROKER_URL=tcps://your-solace-broker-url.com:55443
    SOLACE_BROKER_USERNAME=your-solace-username
    SOLACE_BROKER_PASSWORD=your-solace-password
    SOLACE_BROKER_VPN=your-solace-vpn-name

    # Large Language Model (LLM) API Key
    LLM_API_KEY=your-llm-key
    ```

### Step 2: Build and Run the Container

Once the `.env` file is configured, open a terminal in the project's root directory (`trading_dashboard_demo`) and run the following command:

```bash
docker-compose up --build
```

This command will:
1.  Build the Docker image, installing all Node.js and Python dependencies.
2.  Generate the `solace-ai-connector-config.yaml` file inside the container using your `.env` variables.
3.  Start both the Python `solace-ai-connector` and the React `dashboard` server.

### Step 3: Access the Dashboard

After the build is complete and the containers are running, you can access the trading dashboard in your web browser at:

[http://localhost:5173](http://localhost:5173) 

### Step 4: Update twitter_feed queue

Once the demo is up and running, go to the broker and update the twitter_feed queue (that gets created by AI agent) to respect TTL and set the TTL to 60 seconds. This will ensure that the old tweets (if there is a build up) are regularly cleared.
